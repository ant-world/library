简介

ECMA 和 JavaScript

  • ECMA
    • 欧洲计算机制造协会
  • ECMA-262
    • 欧洲计算机制造协会制定的 标准 ，定义语法和语义
  • ECMAScript
    • 通过 ECMA-262 标准 实现的脚本语言
  • JavaScript
    • 核心使用 ECMAScript 脚本语言，并扩展了其他API功能

JavsScript基础

严格模式

严格模式用法

  1. 脚本文件的顶部定义字符串 “use strict”，整个脚本以严格模式进行运行
  2. 函数中的顶部定义字符串 “use strict”，函数中以喊个模式进行运行

严格模式限制

  1. 不允许使用未定义的变量
  2. 不允许删除变量或对象
  3. 不允许删除函数
  4. 不允许变量重名
  5. 不允许使用八进制
  6. 不允许使用转义字符
  7. 不允许对只读属性赋值
  8. 不允许对一个使用getter方法读取的属性进行赋值
  9. 不允许删除一个不允许删除的属性
  10. 变量名标识符不能为 “eval”
  11. 变量名标识符不能为 “arguments”
  12. 不能使用 with 语句
  13. this 不能指向全局对象，函数中的 this 为 undefined

词法结构

字符集

  1. JavaScript 是用 Unicode 编写的，因此代码中可以使用任何字符
  2. 区分大小写，标识符定义和引用必须一致
  3. JavaScript 会忽略空格和换行符，因此可以随意缩进换行统一编码风格
  4. 无法输入的字符，使用 Unicode转义序列（\uFFFF）

直接量

程序中直接使用的数据

标识符

  1. 标识符就是一个名字，作为变量或函数的名称，或者循环语句跳转位置的标记
  2. 必须以 字母、数字、下划线(_)、美元符($) 进行命名
  3. 必须以 字母、下划线(_)、美元符($) 开始

保留字

JavaScript 预先声明的标识符，用作自己的关键字，程序中不能再重新定义这些标识符

分号

  1. 分号(;) 用于将 语句 分开
  2. 语句独占一行，可以省略分号(;)
  3. 缺少分号(;)，语句作为下一条语句的开始
  4. 缺少了分号(;)的多行语句，上下行语句作为代码无法正常解析时，JavaScript 会自动在两条语句间添加分号(;)
  5. “(“, “[“, “/“, “+”, “-“ 开始的语句极有可能和前一条语句一起解析
    • 常见解决：在他们之前添加 分号(;)
  6. return, break, continue 独占一行，一定会添加分号(;)

术语

  1. 操作数：运算符需要的参数
  2. 一元运算符：只需要一个操作数的运算符
  3. 二元运算符：需要两个操作数的运算符

优先级和结合性

  1. 优先级：表达式有多个运算符，根据优先级决定运算符执行先后顺序
  2. 结合性：相同优先级的运算符在同一个表达式中，且没有括号的时候，运算符和操作数的结合（组合）方式

运算顺序

先计算操作数，再计算运算符：运算符的优先级和结合性决定了复杂表达式的运算顺序，但是没有规定各 操作数（子表达式） 运算的顺序

  1. 操作数（子表达式） 依据从左往右的顺序依次计算
  2. 其值（操作数的值）再依据运算符的优先级和结合性的运算顺序计算复杂表达式

“+” 运算符

连接字符串 “+”

  1. “+” 被用于字符串，会将两个字符串合并连接
  2. 一个操作数是字符串，另一个也将转换为字符串并合并连接
  3. 只有二元运算符 “+” 具有连接字符串效果，其他算术运算符总是将数据转换为 number

转化数字 “+”

  1. 一元运算符 “+”，会将任意类型操作数转换为 nunber，并返回转换后的 number（原始值不变）
  2. 效果等同于 Number(…)

“比较” 运算符

  1. 比较的结果为 Boolean 类型
  2. 字符串比较：是按照字母顺序逐个比较，相同的话比较下一个字母
  3. 不同类型比较：操作数先转换为 number ，然后进行比较
  4. 相等运算符（==）：自动类型转换
  5. 严格相等运算符（===）：不会进行类型转换
  6. undefined、null
    1. undefined == null 值为 true（特例）
    2. undefined === null 值为 false
    3. undefined、null 配合 相等运算符（==），不会转换类型为 number，因此总为 false 【undefined == null 是个特例规定如此】。配合其他操作符则正常转换为 number

“逻辑” 运算符

逻辑与 “&&”

操作数从左往右计算，遇到 false（假值）操作数，直接返回，且不计算右侧操作数

  1. 两个操作数都是布尔值：一个是 false 则返回 false，都是 true 时才返回 true
  2. 操作数是 “真值” 或 “假值”
    1. 操作数按从左往右，总返回第一个 “假值”
    2. 两个操作数都为 “真值”，则返回右边操作数的值

总结：总在寻找 “假值”，碰到 “假值” 就返回

逻辑或 “||”

操作数从左往右计算，遇到 true（真值）操作数，直接返回，且不计算右侧操作数

  1. 两个操作数都是布尔值：一个是 true 则返回 ture，都是 false 时才返回 false
  2. 操作数是 “真值” 或 “假值”
    1. 操作数按从左往右，总返回第一个 “真值”
    2. 两个操作数都为 “假值”，则返回右边操作数的值

总结：总在寻找 “真值”，碰到 “真值” 就返回

逻辑非 “!”

  1. 先将操作数转换为布尔值
  2. 然后取反
  3. 总返回 true/false

“in” 运算符

  1. 属性在指定对象原型链上，返回 true
  2. 不存在的属性，返回 false

“eval” 运算符

  1. 既是运算符，也是全局函数
  2. 运行 JavaScript 代码组成的字符串。先编译，后执行
    1. 编译字符串
      1. 编译失败：抛出语法错误异常
      2. 编译成功：执行此代码
    2. 返回值

变量

声明方式

  2. let

var

  1. 声明变量时没赋值，值为 undefined
  2. 可以重复声明
    1. 重复声明未赋值没任何变化
    2. 重复声明且赋值相当于简单赋值语句
  3. 全局作用域创建的变量，此变量会默认成为全局对象的一个属性
  4. 给未声明变量赋值，此变量会默认成为全局对象的一个属性
  5. 变量提升：变量声明被提前至作用域顶部（只是声明提升到作用于顶部并未赋值，因此值为 undefined）

let

  1. 全局作用域中创建变量，不会作为全局对象的属性
  2. 具有块级作用域
  3. 不能重复声明，包裹用 var/const 声明（let 前后声明），都会报错
  4. 不能变量提升，声明之前访问会报错（处于未初始化状态），而不是 undefined
  5. 全局作用域创建的变量，不作为全局对象属性

常量

声明方式

  • const a = undefined

const

  1. const 声明的变量称为 “常量”
  2. 声明变量时必须赋初始值（初始化变量），否则报错
  3. 初始化的值不能被修改
  4. 具有块级作用域
  5. 不能重复声明，包裹用 var/let 声明（const 前后声明），都会报错
  6. 不能变量提升，声明之前访问会报错（处于未初始化状态），而不是 undefined

变量作用域 / 闭包

变量作用域

什么是变量作用域

程序 源代码中，定义这个变量的区域

变量作用域的作用

限制可用范围：变量在其所在作用域之外不可见（无法被访问），只在其作用域中可以被访问

变量作用域分类

  1. 块级作用域：{ … }
  2. 函数作用域：function(){ … }
  3. 全局作用域：…

只有 let 、const 声明的变量（常量），具有块级作用域，var 声明的变量没有块级作用域

作用域链

基础概念

  1. 词法环境（Lexical Environment）
    1. 特性：
      1. JavaScript 内部（隐藏）的 特殊对象
      2. 函数、代码块{…}、全局环境 都有对应的 词法环境
    2. 组成部分：
      1. 环境记录（Environment Record）：存储对应环境中的变量作为其属性的对象
      2. 外部词法环境的引用：用于引用外部词法环境，从 [[Environment]] 获取
  2. 全局环境没有外部词法环境
  3. [[Environment]]：函数初始化时 自动创建的隐藏属性，保存着创建函数时所在词法环境的引用（外部词法环境的引用）

作用域原理

  1. 变量声明
    1. 所在 环境 开始运行时（函数运行时），对应 词法环境 就会填充对应 作用域 中将会声明的所有变量
      1. 这就是变量提升的原因
      2. var 声明的变量填充后直接进行 初始化
      3. let、const 声明的变量填充后处于 未初始化 状态

  1. 函数声明

    1. 所在 环境 开始运行时，对应 词法环境 就会填充对应 作用域 中将会声明的所有函数（使用语句声明）
    2. 填充后的函数直接 初始化
    3. 函数初始化时，会 创建隐藏属性 [[Environment]]，指向创建函数时所在词法环境（外部词法环境）

  2. 函数调用

    1. 函数调用刚开始时，就会自动创建一个新的词法环境以存储这个调用函数的局部变量和参数
    2. 这个词法环境对外部词法环境的引用从 [[Environment]] 隐藏属性中获取
    3. 当代码要访问一个变量时，首先会搜索内部词法环境，然后搜索外部词法环境，以此类推形成 作用域链

闭包

什么是闭包

闭包：是指一个函数可以记住其外部变量并可以访问这些变量

实现方式

  1. makeCounter() 函数初始化时，具有隐藏属性指向外部词法环境
  2. makeCounter() 被调用时，创建新词法环境，通过隐藏属性指向外部词法环境
  3. 执行到 return 之后，初始化一个匿名函数，此函数具有隐藏属性指向外部词法环境（makeCounter() 被调用时的词法环境）
  4. 调用匿名的函数，创建新的词法环境，并可以引用 makeCounter() 被调用时的词法环境

  1. 整数：
    1. 十进制：0、3、100
    2. 八进制：014、057、077
    3. 十六进制：0xff、0X19AB
  2. 浮点数：
    1. [digits][.digits][(E|e)[(+|-)]digits]
    2. 3.14
    3. .25 // 0.25
    4. 6.02e23 // 6.02 * 10²³
    5. .25E-32 // 0.25 * 10ˉ³²
  3. 全局变量
    1. ±infinity
      1. 正负无穷大
      2. 数字超出 ±1.7976931348623157 * 10³⁰⁸
    2. NaN
      1. 非数字
      2. 与任何数的计算都为 NaN
      3. NaN 与任何数都不相等，包括 NaN 自身

算数运算不报错

  1. “溢出”：返回 ±infinity （正负无穷大）（数字超出 ±1.7976931348623157 * 10³⁰⁸）
  2. “下溢”：返回 0 （数字小于 ±5 * 10ˉ³²⁴）
  3. “…/0”：返回 ±infinity
  4. “0/0”：返回 NaN
  5. 所有不适用于算数运算的操作数，计算结果都将返回 NaN

字符串

基础概念

  1. 使用 UTF-16 编码的 unicode 字符集
  2. 字符串是固定不变的，对字符串的所有操作都是返回新字符串，原字符串不变
  3. 字符串位置从 0 开始
  4. JavaScript 中每个字符由16位内码表示
    1. JavaScript 中字符串是由16位值组成的序列
    2. 对字符串的操作也是基于16位内码进行，而非针对字符进行的操作
    3. 因此字符个数有时不等于字符串长度(length)，一个字符可能由两个16位值组成

  4. 换行定义单行显示：

包装对象

什么是包装对象

存取基本数据类型 “字符串”，“数字”，“布尔值” 的属性时，创建的对象。

包装对象的作用

  1. 使基本数据类型可以像对象一样，调用其属性和方法
  2. 用于区分 “字符串”—“字符串对象”、“数字”—“数值对象”、“布尔值”—“布尔对象”

原理

基本数据类型 “字符串”、“数字”、“布尔值”，使用 . 访问他们的属性，JavaScript 就会将基本类型数据通过调用 new 构造函数转换成对象，通过这个对象调用其属性和方法，属性引用结束，这个对象就会被销毁

特性

  1. 包装对象的属性都是只读的
  2. JavaScript 会在必要时将包装对象转换成原始值
    1. 显示 “==” 比较
    2. 隐式类型转换时

字符串转数字

使用 parseInt()、parseFloat() 全局函数将字符串转换为数字

  1. parseInt()
    1. 根据字符串前缀 “0x”、“0X” 将字符串当作十六进制转换为10进制数字
    2. 会忽略前导空格，尽可能解析数值字符，并忽略后面非数值字符
    3. 第一个非空字符是非数字，则返回 NaN
    4. 第二个参数，指定字符串以相应进制进行解析，返回10进制数字。（取值范围：2~36）
  2. parseFloat()
    1. 会忽略前导空格，尽可能解析数值字符（包括 . 符号），并忽略后面非数值字符
    2. 第一个非空字符是非数字（. 符号除外），则返回 NaN

隐式类型转换

JavaScript 期望使用一种类型数据时，实际可以使用任意类型数据，JavaScript 将根据需要自动转换类型

对象转原始值

  1. 所有对象都继承了 toString()、valueOf() 方法
  2. 本地对象通过 toString()、valueOf() 方法转换为字符串、数值
  3. 宿主对象（例如：web浏览器定义的对象）根据各自算法转换成特定字符串或数字（宿主对象有自己特有的字符串、数值）
  4. 本地对象：ECMAScript 实现中，并非由宿主环境，而是完全由本规范定义其语义的对象

对象转布尔值

都是 true

对象转字符串

宿主对象 toString() 返回值：

  1. 数组：元素转换为字符串，元素间添加逗号后合并起来的字符串
  2. 函数：函数定义的字符串形式
  3. 日期类：可读的日期和时间字符串
  4. RegExp：返回正则字面量字符串形式

本地对象转字符串过程：

  1. 尝试调用 toString() 方法
  2. 如果返回原始值，原始值转换为字符串并返回
  3. 没有 toStrinig() 方法，或者没有返回原始值，则调用 valueOf() 方法
  4. 如果返回原始值，原始值转换为字符串并返回
  5. 如果 toString()、valueOf() 都无法返回原始值，抛出类型错误

对象转数值

宿主对象 valueOf() 返回值：

  1. 日期类：返回1970年1月1日以来的毫秒数
  2. 包装对象：valueOf() 返回其对应的原始值
  3. 本地对象：valueOf() 默认返回对象本身

本地对象转数字过程：

  1. 尝试调用 valueOf() 方法
  2. 如果返回原始值，原始值转换为数字并返回（本地对象默认返回对象本身）
  3. 没有 valueOf() 方法，或者没有返回原始值，则调用 toString() 方法
  4. 如果返回原始值，原始值转换为数字并返回
  5. 如果 toString()、valueOf() 都无法返回原始值，抛出类型错误
